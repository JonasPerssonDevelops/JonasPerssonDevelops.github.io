    Slack Web Widget Prototype May 2022

    Background

    A member of upper management at 4G Clinical had an idea one day that it would be useful to integrate all the web services that they were using which were Jira project tracking management, Slack messaging, Google Oauth, and ZenDesk ticket tracking. And the chief reason to do this was because currently the main means of communication with clients was email.

    Integrating Slack messaging; a calender with important dates and deadlines; and a timeline with major phases together on the same page would make project management nearly effortless. It would make it possible for a company employee or client to communicate without digging through emails, looking on their computer for the relevant files, and using email as a messaging tool. It would also allow clients to access the tools and project specific data the company was currently using so any changes to projects would be immediately reflected to clients, with data from different services all on a single page. This would free the company from sending mountains of emails with deadline reminders, documents, and resending documents that got lost, instead deadline changes could be done just by changing them in the project management tools and have notifications sent automatically to the relevant employees and clients.

    The Slack Web Widget

    In the subteam I lead we were responsible for developing the Slack messaging capabilities for the web app. I took responsibility for the frontend and the REST services for the widget and worked on parts of the backend as well.

    The architecture of the web widget spanned across three major areas:

    • Frontend: HTML, CSS styling, and JavaScript
    • Web API / REST services: GETs and POSTs
    • Backend: functions that fetch and send data to Slack's API and that process the return data

    The widget was programmed from scratch without any HTML templates, styling frameworks, or predefined REST implementations.

    I designed a custom Web API for the widget consisting of a GET service that called functions on the backend to retrieve the chat message history for different channels. User messages were handled with a POST that was done through JavaScript with jQuery.

    A World First

    This widget also features a world first. A user can check the All checkbox, an action only requiring a couple of seconds of user interaction, to send a message to multiple Slack channels. Slack provides a similar functionality themselves which requires the user to fill out a form and pick a particular time to post a message to multiple channels. Our teams version accomplishes the same task with only a couple of seconds of user interaction required.

    AppointmentBookster Android App August 2021
    -An app for keeping track of appointments

    Background

    This was my implementation of the final project in the CS410 Java Intensive Programming class during Summer 2021.

    Development

    This app was developed in Java from a template in the Android Studio IDE. The UI was built from scratch with the exception of the top navigation bar and the snackbar popup feature of Android.

    Features

    The app can store appointments with the three following pieces of detail: description, start date and time, and end date and time. Multiple users can be created to store appointments under and as such it can function as an appointment book for a secretary who must manage multiple clients.

    Snackbar pop-ups notify the user when input errors occur and inform them when major actions have been successfully completed, such as successful entry of a new appointment or whether appointments were found within the timeframe being searched for.

    The app features data persistence. Each new appointment is written to a file in the directory allocated to it in it's sandbox.
